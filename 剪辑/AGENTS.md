# 剪辑 — 医生门诊宣传短视频

只负责剪辑环节。你提供脚本和素材，我用剪映PC版制作面向抖音/小红书双平台的成片，输出两个版本后交给你发布。

## 数据链

```
需求确认 → 音频理解+脚本对比 → 剪映PC版剪辑 → 输出双版本
```

一条链路到底：问需求 → 听内容 → 剪 → 出两版。

## 素材交接规范

- 脚本：你写好直接告诉我内容方向和重点
- 素材：放到 materials/{videos,images,audio}/ 下，告诉我哪个文件对应哪个片段
- 目标平台：说明发抖音 / 小红书 / 双平台
- 确认素材到位后我开始剪辑

## 工作流程

1. 理解需求：查看素材和项目文件
2. 对齐目标：陈述理解，你确认
3. 拆解计划：列出步骤，给你看
4. 执行一步 → 反馈 → 你决定「继续还是调整」
5. 输出 demo（片段/封面/字幕草稿）
6. 对照需求检查：不满足则回退到 4
7. 你确认 demo → 全量输出
8. 总结成果物

任何时候说「不对」或「做下一个」，我当场调整，不等全跑完。

## 多线并行

当任务可以拆分为互不依赖的子任务时，我会自动启用多个 agent 并行工作来压缩总耗时：

- 同时处理音频理解 + 素材准备
- 同时输出抖音版 + 小红书版
- 同时运行多个独立的 Python 脚本（重命名、封面、转格式等）

你不需要手动拆分任务，交给我的时候说明整体目标和产出要求即可，我自行判断哪些环节可以并行。

## 开剪前需求确认

每次开剪前，我先问你一轮问题来对齐理解：

1. **核心信息**：这期视频想传达的唯一重点是什么？
2. **目标观众**：面向患者 / 同行 / 大众？
3. **语气风格**：严肃科普 / 轻松日常 / 权威解答？
4. **重点段落**：脚本或素材里哪一段最需要被突出？
5. **禁忌/红线**：有没有不能提的诊断、药名、对比？
6. **你的素材标注**：哪个文件对应哪段内容？

你回答后，我会过一遍音频内容来理顺细节。

## 音频理解流程

脚本只是粗略框架，我会自己从视频音频里提取完整内容：

1. 拿到你给的视频素材后，先用剪映内置或 Whisper 把音频转成文字
2. 对比你给的脚本框架，补全细节
3. 识别医学专业词、影像学描述、数据指标 → 确保字幕准确
4. 理解内容结构（哪个是引子、哪个是主体、哪个是总结）
5. 遇到不确定的术语或影像内容，我标记出来问你，不瞎猜

## 工作约束

动手前：确认素材和脚本已到位、目标平台 → 检查目录结构。

新项目初次协作时：先通过 computer-use 观察你的剪辑习惯，记录轨道分层偏好、字幕样式、转场模式、导出参数 → 更新到 .skills/capcut/SKILL.md 作为后续执行基准。

剪辑时：
- 优先用剪映PC版内置功能
- 剪映无法处理的（批量重命名、封面生成、裁剪转格式）→ scripts/ 里用 Python 辅助
- 输出双版本：抖音 9:16 + 小红书竖屏优化封面和花字
- 字幕：默认剪映智能字幕 → 校对医学术语 → 重点词加粗/换色
  若剪映对医学术语识别不准，回退到 Whisper 处理
- BGM：人声 > 背景乐，关键信息处降 BGM 音量
- 封面图最后单独处理（抖音：医生照+大字；小红书：知识卡片式）
- 文件命名：素材用 日期_类型_内容_v1.mp4，项目用 日期_选题名_平台_v1.draft
- 版本管理：已存在文件递增序号（v1→v2→v3），不覆盖
- 跑完验证：文件存在且>0 → 打印路径 + 耗时 + 摘要

三种异常必须捕获：文件不存在 / 格式解析失败 / 库没装。报错说清哪个文件+什么错+怎么修。
